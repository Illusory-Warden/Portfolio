Description

AroGuard is an AI-powered web application designed to help users make informed decisions about food products by decoding complex ingredient lists, E-numbers, and preservatives. The application leverages Groq AI to provide real-time safety analysis and comprehensive ingredient information, making it easier for health-conscious consumers to understand what they're eating.

Built with modern web technologies including React and TypeScript, AroGuard combines multiple data sources and AI capabilities to deliver a seamless user experience with instant ingredient analysis, allergen detection, and interactive visualizations.

Key Features & Achievements

  • AI-Powered Analysis: Integrated Groq AI API for intelligent ingredient decoding and safety classification
  • Real-Time Safety Classification: Instant categorization of ingredients into safe, caution, and harmful categories
  • Interactive Visualizations: Dynamic pie charts using Recharts library to display ingredient safety breakdown
  • Barcode Search: Quick product lookup using barcode scanning functionality
  • Open Food Facts Integration: Access to comprehensive product database with AI fallback scraping
  • Allergen Detection: Automatic identification of common allergens in ingredient lists
  • IndexedDB Caching: Offline-first architecture with local data persistence for faster repeat searches
  • AI Chatbot: Interactive assistant for answering ingredient-related questions
  • Type-Safe Development: Built with TypeScript for enhanced code quality and maintainability

Technical Challenges & Solutions

Challenge 1: Reliable Data Sourcing

Food product databases can be incomplete or outdated. Relying solely on Open Food Facts API would result in missing data for many products, especially regional or newly launched items.

Solution: Implemented a hybrid data strategy combining Open Food Facts API as the primary source with AI-powered web scraping as a fallback. The Groq AI model intelligently extracts ingredient information from product websites when API data is unavailable, ensuring comprehensive coverage.

Challenge 2: Real-Time AI Processing Performance

Processing ingredient lists through AI models can introduce latency, creating a poor user experience. Users expect instant results when analyzing products.

Solution: Implemented IndexedDB caching to store previously analyzed products locally. Combined with the fast response times of Groq AI (optimized for speed), the application delivers near-instant results for repeat searches while maintaining quick first-time analysis.

Challenge 3: Complex Ingredient Classification

Food ingredients can have complex names, E-numbers, and varying safety profiles depending on concentration and individual sensitivities. Creating accurate classifications requires understanding nuanced health data.

Solution: Leveraged Groq AI's natural language understanding to analyze ingredients in context. The AI considers multiple factors including regulatory status, common allergies, concentration levels, and recent research to provide nuanced safety classifications rather than binary safe/unsafe ratings.
